# PRODIOSLABS | Data Analyst Assignment

## Exam Behaviour Analysis using PostgreSQL & Power BI

## Project Overview

This project analyzes candidate behaviour during an online examination using event-level telemetry logs. The dataset contains over 8 million event records generated by approximately 88,000 candidates.

The objective of this analysis was to examine:

* Candidate participation patterns
* Activity distribution
* Review behaviour
* Language distribution
* Response-state trends
* Section-wise interaction volumes
* Session duration
* Hourly event trends

The analysis was performed using PostgreSQL for data processing and Power BI for dashboard development.

## Composite Key Validation

To ensure record uniqueness, the composite key:

(log_id, candidate_id)

was validated using:

```sql
SELECT
    log_id,
    candidate_id,
    COUNT(*)
FROM candidate_log
GROUP BY log_id, candidate_id
HAVING COUNT(*) > 1;
```

Result:

* No duplicate composite-key combinations were identified.

# Dashboard Overview

## Page 1 – Executive Overview

### Visuals

* Total Candidates KPI
* Total Events KPI
* Auto Save Percentage KPI
* Active Interaction Percentage KPI
* Candidate Engagement Segmentation
* Activity Distribution

### Key Findings

* More than 88,000 candidates participated in the examination.
* Auto Save events accounted for approximately 90.73% of all recorded events.
* Manual interactions represented approximately 9.27% of recorded events.
* Candidate segmentation is heavily influenced by Auto Save activity and should be interpreted as event-volume categories rather than direct measures of engagement.

---

## Page 2 – Exam Behaviour Analysis

### Visuals

* Section-wise Language Distribution
* Response-State Distribution
* Review Behaviour Analysis

### Key Findings

* English interactions exceeded Hindi interactions across all sections.
* Language distribution varied across examination sections.
* Mark for Review actions occurred more frequently than UnMark actions.
* Section 1 generated the highest interaction volume.
* Option A appeared most frequently in recorded response states, while Option D appeared least frequently.

### Important Note

Due to the dominance of Auto Save events and the absence of an answer key, response-state distributions should not be interpreted as:

* Candidate preference
* Answer popularity
* Answer correctness

---

## Page 3 – Time Analysis

### Visuals

* Hourly Event Trend
* Session Duration Analysis

### Key Findings

* Event volume peaked around 16:00.
* Elevated event activity was observed around 09:00 and 12:00–13:00.
* Event activity declined significantly after 17:00.

### Interpretation

Hourly trends represent overall event-volume patterns and include substantial Auto Save activity. These trends should not be interpreted as direct measures of candidate behaviour.

# DAX Measures

## Total Candidates

```DAX
Total Candidates =
SUM(candidate_segmentation[candidates])
```

## Total Events

```DAX
Total Events =
SUM(activity[total_events])
```

## Auto Save Percentage

```DAX
Auto Save % =
DIVIDE(
    CALCULATE(
        SUM(activity[total_events]),
        activity[activity] = "Auto Save"
    ),
    SUM(activity[total_events])
)
```

## Active Interaction Percentage

```DAX
Active Interaction % =
1 - [Auto Save %]
```

# Assumptions & Limitations

* Auto Save events were treated as system-generated interactions.
* Approximately 90.73% of records are Auto Save events.
* No answer key was available.
* Event volume does not necessarily indicate candidate engagement.
* Response-state distributions should not be interpreted as correctness or popularity.
* Hourly event peaks may be influenced by system-generated activity.

# Conclusion

The examination generated a substantial volume of event data, with Auto Save activity accounting for most recorded events. Review behaviour, language distribution, response-state analysis, session duration metrics, and hourly event trends provide meaningful insight into examination activity. The analysis highlights the importance of separating candidate-driven actions from system-generated events to ensure accurate interpretation of behavioural patterns.
